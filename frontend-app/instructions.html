Simple Guide for Non-Technical Users

This guide will help you set up and use the Video Processing System with your AWS account. We'll walk through each step in simple terms.

1Create an AWS Account

If you don't already have an AWS account, you'll need to create one:

  1. Go to aws.amazon.com
  2. Click "Create an AWS Account" button in the top right
  3. Follow the instructions to create your account
  4. You'll need to provide a credit card, but many AWS services have a free tier

Note: AWS will charge you for the resources you use. Make sure to monitor your usage to avoid unexpected charges.

2Create an S3 Bucket

S3 is where your videos will be stored:

  1. Sign in to the AWS Management Console
  2. Search for "S3" in the search bar at the top
  3. Click on "S3" in the search results
  4. Click the "Create bucket" button
  5. Enter a unique bucket name (like "my-video-bucket-123")
  6. Select your preferred region (e.g., "ap-south-1")
  7. Leave all other settings as default
  8. Scroll down and click "Create bucket"

Remember: Write down your bucket name - you'll need it later!

3Create IAM User and Get Access Keys

You need access keys to let the application use your AWS account:

  1. In the AWS Console, search for "IAM" and click on it
  2. In the left sidebar, click "Users"
  3. Click "Create user"
  4. Enter a name like "video-transcoder-user"
  5. Click "Next"
  6. Select "Attach policies directly"
  7. Search for and select these policies:
    • AmazonS3FullAccess
    • AmazonECSFullAccess
    • CloudWatchLogsFullAccess
  8. Click "Next" and then "Create user"
  9. Click on the user you just created
  10. Go to the "Security credentials" tab
  11. Scroll down to "Access keys" and click "Create access key"
  12. Select "Application running outside AWS"
  13. Click "Next" and then "Create access key"
  14. IMPORTANT: Download the CSV file or copy both the Access Key ID and Secret Access Key - you will need these later and won't be able to see the Secret Access Key again!

Warning: Never share your Secret Access Key with anyone. It gives access to your AWS account and resources.

4Create an ECS Cluster

ECS is the service that will run the video transcoding:

  1. In the AWS Console, search for "ECS" and click on it
  2. Click "Clusters" in the left sidebar
  3. Click "Create Cluster"
  4. Enter "video-transcoder-cluster" as the cluster name
  5. Under "Infrastructure", select "AWS Fargate (serverless)"
  6. Leave all other settings as default
  7. Click "Create"

Note: After creating the cluster, you'll see an ARN (Amazon Resource Name) that looks like: arn:aws:ecs:ap-south-1:123456789012:cluster/video-transcoder-cluster. Write this down - you'll need it later.

5Create a Task Definition

This tells AWS how to run the transcoding software:

  1. In the ECS console, click "Task Definitions" in the left sidebar
  2. Click "Create new Task Definition"
  3. Select "Fargate" as the launch type
  4. Click "Next step"
  5. Enter "video-transcoder" as the task definition name
  6. For "Task Role", select "ecsTaskExecutionRole" (create it if it doesn't exist)
  7. For "Task execution role", select "ecsTaskExecutionRole"
  8. Set "Task memory" to "4GB"
  9. Set "Task CPU" to "2 vCPU"
  10. Click "Add container"
  11. Enter "video-transcoder" as the container name
  12. Enter "767828740984.dkr.ecr.ap-south-1.amazonaws.com/sasa-trnscd:latest" as the image
  13. Set "Memory Limits" to "4096"
  14. Leave "Port mappings" empty
  15. Under "Environment variables", add:
    • Name: AWS_REGION, Value: your region (e.g., ap-south-1)
    • Name: AWS_ACCESS_KEY_ID, Value: your access key
    • Name: AWS_SECRET_ACCESS_KEY, Value: your secret key
    • Name: BUCKET_NAME, Value: your S3 bucket name
  16. Click "Add"
  17. Click "Create"

Note: After creating the task definition, note the name and revision number (e.g., "video-transcoder:1"). You'll need this later.

6Get Your VPC Information

You need subnet and security group information for the application:

  1. In the AWS Console, search for "VPC" and click on it
  2. In the left sidebar, click "Subnets"
  3. Note the IDs of at least two subnets (they look like "subnet-1234abcd")
  4. In the left sidebar, click "Security Groups"
  5. Note the ID of the default security group (it looks like "sg-1234abcd")

Note: Write down the subnet IDs and security group ID - you'll need these for configuration.

7Configure the Application

Now you'll set up the application with all the information you've gathered:

  1. Go to the Configuration page: Open Configuration
  2. Fill in the following details:
    • AWS Region: Your AWS region (e.g., ap-south-1)
    • AWS Access Key ID: Your IAM user's access key
    • AWS Secret Access Key: Your IAM user's secret key
    • S3 Bucket Name: The name of your S3 bucket
    • ECS Cluster: The ARN of your ECS cluster (e.g., arn:aws:ecs:ap-south-1:YOUR-ACCOUNT-ID:cluster/video-transcoder-cluster)
    • ECS Task Definition: Your task definition name and revision (e.g., video-transcoder:1)
    • ECS Subnets: Your subnet IDs, separated by commas (e.g., subnet-1234abcd,subnet-5678efgh)
    • ECS Security Groups: Your security group ID (e.g., sg-1234abcd)
  3. Important: Make sure your subnets are in a VPC with internet access through a NAT Gateway or Internet Gateway, as the application uses Fargate launch type which requires internet access.
  4. Click "Test Connection" to verify your configuration
  5. If the test is successful, click "Save Configuration"

Note about Subnets: For Fargate tasks, your subnets must have internet access. If you're using private subnets, they must have a route to a NAT Gateway. If you're using public subnets, they must have a route to an Internet Gateway.

1Upload a Video

  1. Go to the Home page: Open Home
  2. Click "Choose File" to select a video file from your computer
  3. Click "Upload" to upload the video to your S3 bucket
  4. Wait for the upload to complete - you'll see a success message when it's done
  5. Once the upload is complete, click "Start Transcoding"

Note: The maximum file size is 500MB. Larger files may fail to upload.

2Monitor Your Transcoding Job

  1. After starting a transcoding job, you'll be automatically redirected to the Jobs tab
  2. Here you can see the status of all your transcoding jobs
  3. The status will be one of:
    • PENDING: The job is waiting to start
    • RUNNING: The job is currently processing your video
    • COMPLETED: The job has finished successfully
    • FAILED: The job encountered an error
  4. Click on a job to view its details and logs
  5. The status will update automatically as the job progresses

Note: Transcoding can take several minutes depending on the size and length of your video.

3View Your Transcoded Videos

  1. Once a job is completed, you can view the transcoded video
  2. Go to the Videos tab
  3. Select the video you want to watch
  4. Use the quality selector to switch between different resolutions (360p, 480p, 720p, 1080p)

Note: The transcoded videos are stored in your S3 bucket in the output/[video-id]/ directory. The HLS playlist is at output/[video-id]/master.m3u8.

Common Errors and Solutions

Important: AWS Account ID Consistency

Many errors occur because of AWS account ID mismatches. Your AWS account ID must be consistent across:

  • The AWS credentials (Access Key and Secret Key) you're using
  • The ECS cluster ARN in your configuration
  • The task definition ARN (if using the full ARN)
  • The ECR repository in your task definition's container image

Your AWS account ID is a 12-digit number (e.g., 767828740984) that you can find by clicking your username in the top-right corner of the AWS Console.

Error: "Account ID mismatch"

This error occurs when the AWS account ID in your ECS cluster ARN doesn't match the account ID associated with your AWS credentials.

Solution:

  1. Find your correct AWS account ID:
    • Log into the AWS Console
    • Click on your username in the top-right corner
    • Your 12-digit account ID is displayed in the dropdown (e.g., 767828740984)
  2. Update your ECS cluster ARN in the configuration:
    • Go to the Configuration page
    • Find the ECS Cluster field
    • Make sure it uses your correct account ID: arn:aws:ecs:ap-south-1:YOUR-ACCOUNT-ID:cluster/video-transcoder-cluster
    • Replace YOUR-ACCOUNT-ID with your actual 12-digit account ID
  3. Make sure your AWS credentials (Access Key and Secret Key) belong to the same AWS account
  4. Click "Test Connection" to verify the fix

Important: The account ID in your ECS cluster ARN must match the account ID of the AWS credentials you're using.

Error: "SignatureDoesNotMatch: The request signature we calculated does not match the signature you provided"

This error means there's a problem with your AWS credentials.

Solution:

  1. Double-check that you've entered your AWS Access Key ID correctly
  2. Make sure your AWS Secret Access Key is correct (no extra spaces)
  3. Check that your computer's date and time are accurate
  4. Try creating new access keys in the IAM console

Error: "Invalid subnet configuration" or "Invalid security group configuration"

This error occurs when the subnet IDs or security group ID you provided don't exist or aren't accessible.

Solution:

  1. Go to the AWS VPC console and verify that the subnet IDs exist
  2. Check that the security group ID exists
  3. Make sure all subnets and the security group are in the same VPC
  4. Ensure you have at least two subnets in different availability zones
  5. Update your configuration with the correct IDs

Error: "Failed to upload video"

This error occurs when the video upload to S3 fails.

Solution:

  1. Check that your S3 bucket exists and is accessible
  2. Verify that your AWS credentials have permission to write to the bucket
  3. Ensure the video file is not too large (limit is 500MB)
  4. Try uploading a smaller video file to test

Error: "Failed to start transcoding"

This error occurs when the transcoding job fails to start.

Solution:

  1. Check the job logs for detailed error messages
  2. Verify that your ECS cluster and task definition are correctly configured
  3. Make sure your IAM user has the necessary permissions
  4. Check that your subnets have internet access (for Fargate tasks)
  5. Verify that the task definition uses the correct container image

Error: "AWS account is blocked"

This error occurs when there's an issue with your AWS account status.

Solution:

  1. Check your AWS account status in the AWS Console
  2. Verify that your payment method is valid
  3. Contact AWS Support if you believe your account was blocked in error

Error: "Configuration missing"

This error occurs when you try to use the application before configuring it.

Solution:

  1. Go to the Configuration page
  2. Fill in all the required fields
  3. Click "Save Configuration"
  4. Try your operation again

Error: "Task Definition Account ID mismatch"

This error can also occur when the account ID in your task definition doesn't match your AWS credentials.

Solution:

  1. In the Configuration page, check your Task Definition field
  2. If you're using a full ARN (like arn:aws:ecs:ap-south-1:767828740984:task-definition/video-transcoder:1), make sure the account ID matches your AWS credentials
  3. The simplest solution is to use just the task definition name and revision instead of the full ARN:
    • Use: video-transcoder:1
    • Instead of: arn:aws:ecs:ap-south-1:767828740984:task-definition/video-transcoder:1
  4. Click "Test Connection" to verify the fix

Error: "Container Image Account ID mismatch"

This error can occur when the account ID in the container image URL doesn't match your AWS credentials.

Solution:

  1. When creating your task definition, check the container image field
  2. The image URL should be: 767828740984.dkr.ecr.ap-south-1.amazonaws.com/sasa-trnscd:latest
  3. If you're using your own AWS account, you need to:
    • Create your own ECR repository named "sasa-trnscd"
    • Push the image to your repository
    • Update the image URL to use your account ID: YOUR-ACCOUNT-ID.dkr.ecr.ap-south-1.amazonaws.com/sasa-trnscd:latest
  4. If you're using the provided account, make sure your AWS credentials match the account ID in the image URL

Error: "Assign public IP is not supported for this launch type"

This error occurs when trying to run an ECS task with a network configuration that's incompatible with your launch type.

Solution:

  1. Go to the Configuration page
  2. Make sure your ECS task definition is using the correct launch type:
    • For EC2 launch type: Remove the "assignPublicIp" setting or set it to "DISABLED"
    • For Fargate launch type: Make sure your subnets have internet access through a NAT Gateway or Internet Gateway
  3. If you're using Fargate (recommended), update your configuration:
    • Make sure your VPC has a NAT Gateway or Internet Gateway
    • Ensure your subnets are properly configured for public IP assignment
    • In the Configuration page, add FARGATE as the Launch Type
  4. If you're using EC2, update your task definition to use EC2 launch type and disable public IP assignment

Error: "Task stuck in PROVISIONING state"

This error occurs when an ECS task gets stuck in the PROVISIONING state and doesn't progress to RUNNING.

Solution:

  1. Check your VPC and subnet configuration:
    • Make sure your subnets have internet access (either through NAT Gateway or Internet Gateway)
    • Verify that your security groups allow outbound traffic
    • Ensure your VPC has DNS resolution enabled
  2. Check your ECR repository permissions:
    • Make sure your task execution role has permission to pull from ECR
    • Verify that the image exists in the specified ECR repository
  3. Check ECS service quotas:
    • You might have reached your Fargate quota limit
    • Go to Service Quotas in the AWS Console to check your limits
  4. Try stopping the task and starting a new one

Understanding ECS Task States:

  • PROVISIONING: AWS is preparing resources for your task
  • PENDING: Resources are ready, but the container is not yet started
  • RUNNING: The container is running and executing your application
  • STOPPED: The task has completed or failed
