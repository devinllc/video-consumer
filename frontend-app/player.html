Important Note About Transcoding

When deployed on Vercel or similar serverless platforms, actual video transcoding cannot be performed due to serverless environment limitations:

  • Serverless functions have limited execution time (typically 10-60 seconds)
  • Serverless environments don't support long-running processes
  • Memory and CPU resources are limited

To fully use video transcoding features:

  • Deploy this application on a traditional server (AWS EC2, DigitalOcean, etc.)
  • Use AWS Lambda with extended timeout for moderate transcoding tasks
  • Use AWS Batch or AWS ECS directly for heavy transcoding workloads

The current deployment simulates job creation but doesn't actually process videos.
